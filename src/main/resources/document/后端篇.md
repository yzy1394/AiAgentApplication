# 后端篇：常见问题与回答

## 1. Java 后端项目如何做分层设计？
推荐按 controller、service、repository、domain 分层，控制器只处理协议转换，业务规则放在 service，数据访问聚焦在 repository。

## 2. 如何提升 Spring Boot 接口性能？
优先做数据库索引优化、减少 N+1 查询、增加热点缓存（Redis）、并对慢接口做异步化和批处理。

## 3. 事务应该怎么用？
以业务一致性边界定义事务，避免大事务；读写分离场景下明确传播行为，关键链路补充重试与幂等控制。

## 4. 如何设计可扩展的 API？
使用清晰资源模型和版本策略（如 /v1），保持向后兼容；对分页、过滤、排序参数做统一规范。

## 5. 如何排查线上故障？
先看监控（QPS、RT、错误率），再结合日志和链路追踪定位瓶颈，最后通过灰度和回滚控制风险。
